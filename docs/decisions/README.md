# Architecture Decision Records

This directory contains Architecture Decision Records (ADRs) documenting key technical decisions for the standards repository.

## Active ADRs

- [ADR-ACCURACY-CLEANUP.md](ADR-ACCURACY-CLEANUP.md) - Decision to prioritize accuracy over marketing language (2025-10-25)

## ADR Template

Use the format from ADR-ACCURACY-CLEANUP.md as a template for new decisions:
- Context & drivers
- Options considered
- Decision outcome
- Consequences (positive & negative)
- Validation criteria

## Purpose

ADRs help us:
- Document why decisions were made
- Provide context for future maintainers
- Enable informed decision reversals
- Track architecture evolution
